# Verbal Paradigms: The Complex Future Tense

## Introduction
While simpler paradigms can be handled with direct feature mapping, Irish verbal morphology often requires complex interactions between stem classes, tense markers, and phonological rules. In this chapter, we explore a complex use-case: the **Irish Future Tense**, contrasting the 1st and 2nd conjugations.

## Verbal Feature Space and Tags

To model the full complexity of Irish verbs, we define a multi-dimensional feature space. Each tag corresponds to a specific grammatical axis that triggers distinct morphological and phonological outcomes.

| Feature | Code | Description | Valid Values |
| :--- | :--- | :--- | :--- |
| **Conjugation** | `conj` | The verbal inflectional class. | `C1`, `C2` |
| **Tense** | `tense` | The time of the action. | `pres`, `past`, `fut`, `imperf` |
| **Mood** | `mood` | The modality of the action. | `ind`, `imp`, `subj`, `cond` |
| **Person** | `person` | The grammatical person. | `P1`, `P2`, `P3`, `Pauto` |
| **Number** | `number` | The grammatical number. | `sg`, `pl` |
| **Mutation** | `mut` | Initial consonant mutation. | `none`, `len`, `ecl` |

By instantiating a `FeatureVector` for every valid combination of these codes, the eGrammar creates a deterministic map that guides the FST through the correct sequence of suffixation and stem-internal modifications.


## Modeling Conjugation Classes

The Irish future tense is defined by two distinct markers:
- **1st Conjugation**: Uses the marker `-f-` (e.g., *mol* → *molfaidh*).
- **2nd Conjugation**: Uses the marker `-ó-`/`-eó-` and often involves **syncope** or the deletion of the `-igh` terminal (e.g., *ceannaigh* → *ceannóidh*).

### 1. Feature Representation
We define features to distinguish between conjugation classes and specific grammatical slots.

```python
conjugation = features.Feature("conj", "C1", "C2")
tense = features.Feature("tense", "fut")
verb_cat = features.Category(conjugation, tense, person, number)
```

### 2. Rule-Based Stem Transformation
The 2nd conjugation is particularly interesting because it requires a stem transformation (deleting `-igh`) *before* the affixation is realized. This demonstrates the power of cascading FSTs in an item-and-process model.

```python
# 1. Stem Modification: Delete 'igh' before the 2nd conjugation marker (+O)
delete_igh = pynini.cdrewrite(
    pynini.cross("igh", ""), 
    "", 
    "+O", 
    sigma_star
)

# 2. Suffix Realization: Convert abstract markers to surface orthography
realize_f = pynini.cdrewrite(pynini.cross("+F", "faidh"), "", "", sigma_star)
realize_o = pynini.cdrewrite(pynini.cross("+O", "óidh"), "", "", sigma_star)

# The final cascade handles the logical sequence of changes
future_rules = (delete_igh @ realize_f @ realize_o).optimize()
```

### 3. Integrating with the Paradigm
By using abstract markers like `+F` and `+O` in the morphological slots, we can define a single `Paradigm` object that handles both conjugation classes based on the input features.

```python
slots = [
    (paradigms.suffix("+F", stem), fut_1_3sg), # 1st Conj
    (paradigms.suffix("+O", stem), fut_2_3sg), # 2nd Conj
]

v_paradigm = paradigms.Paradigm(
    category=verb_cat,
    slots=slots,
    rules=[future_rules]
)
```

The manual evaluation for the verb-pradigm for the verb <ga ipa="/mˠɔlˠ/">mol</ga> is shown next. For the Irish verb **mol** (to praise), which is a 1st Conjugation verb, the **Future tense** provides a perfect example of how morphophonology and phonotactics interact to create the surface form.

The resulting surface form for the **1st Person Singular Future** is:

### **Molfaidh mé** (or **Molfad**)

Depending on whether you use the analytic or synthetic form, here is how the pipeline produces that result.

#### 1. Morphological Stage (The Input)

You start with the root and the "Future" grammatical marker.

* **Root:** *mol* (praise)
* **Future Marker:** *-f-* (The characteristic marker for 1st conjugation future)
* **1st Person Singular Ending:** *-aidh* (analytic) or *-ad* (synthetic)

#### 2. Morphophonological Stage (Early)

The system selects the correct future suffix based on the conjugation class. For *mol*, it selects the **-f-** suffix.

* **Intermediate string:** *molf...*

#### 3. Phonotactic Stage (Late - The "Legal" Shape)

This is where the **Vowel Harmony** (Phonotactics) rule determines the final spelling. Because the last vowel in the root *m**o**l* is **broad** (*a, o, u*), the entire suffix must also be broad.

* **The Constraint:** *Caol le caol agus leathan le leathan* (Slender with slender, broad with broad).
* **The Adjustment:** The system chooses the "broad" version of the endings:
* **Analytic:** *mol* + *faidh* → **Molfaidh mé**
* **Synthetic:** *mol* + *fad* → **Molfad**



---

### Comparison: Broad vs. Slender Phonotactics

If the root had a slender vowel, the phonotactics would force a different surface form:

| Verb Root | Type | Morphological Marker | Phonotactic Adjustment | Surface Form |
| --- | --- | --- | --- | --- |
| **Mol** (Praise) | Broad | *-f-* | **-faidh** (Broad) | **Molfaidh** |
| **Cuir** (Put) | Slender | *-f-* | **-fidh** (Slender) | **Cuirfidh** |

### The FST Perspective

In your computational linguistics work, your FST would likely treat the future suffix as an abstract morpheme `^FUT`. The transducer would then have two paths (allomorphs) for that morpheme:

1. `^FUT` → `fidh` / `SlenderVowel` _
2. `^FUT` → `faidh` / `BroadVowel` _

The **Phonotactic layer** is what actually "decides" which path the transducer takes based on the vowel environment of the root.
