# Phonetic Generation: Translating Spelling to Sound

If you have ever tried to learn Irish, you have likely run headfirst into one of
its most notorious hurdles: the spelling.

To an English speaker, Irish orthography (how words are spelled) can seem
incredibly intimidating. There appear to be far too many vowels clustered
together, and strings of consonants—like <span class="ga-word">bhf<span class="ga-tooltip">/w/</span></span> or <span class="ga-word">mh<span class="ga-tooltip">/w/</span></span> —that simply don't make
sense to an English-trained eye.

In fact, this difficulty is a running joke on international television. Hosts
will often present a deceptively complicated-looking Irish name or word to a
non-Irish speaker and ask them to read it aloud. After the guest struggles
through a clumsy, literal pronunciation (guessing "See-oh-ba-han?"), an Irish
speaker reveals the actual, effortlessly smooth pronunciation ("Shiv-awn"), to
the great hilarity of the audience.

While it makes for great comedy, this disconnect between spelling and
pronunciation is a massive barrier for language learners. But here is the
secret: Irish spelling isn't actually random. It is highly systematic; it is
just following a completely different, much older rulebook than English.

## Rewriting Strings: From Spelling to IPA

Because Irish spelling follows strict mathematical rules (even if they seem
convoluted to beginners), we can use a **Finite State Transducer (FST)** to
crack the code.

As we learned in the previous chapter, an FST is a computational machine
designed to perfectly "rewrite" one string of text into another based on a
grammar rulebook. For phonetic generation, our goal is to rewrite the **written
Irish characters** into the **International Phonetic Alphabet (IPA)**.

The IPA is a globally standardized set of symbols where every character
represents one specific, unambiguous sound.

Let's look at how wildly different the Irish spelling is from the actual
phonetic sounds it represents:

- <span class="ga-word">Siobhán<span class="ga-tooltip">/ʃʊwəːnˠ/</span></span> (a common name)
  - **IPA Translation:** `/ˈʃʊ.waːn̪ˠ/` (Pronounced roughly like Shuv-awn)
- <span class="ga-word">Taoiseach<span class="ga-tooltip">/t̪ˠɪːʃaç/</span></span> (Prime Minister)
  - **IPA Translation:** `/ˈt̪ˠiːʃəx/` (Pronounced roughly like Tee-shuck)
- <span class="ga-word">Samhradh<span class="ga-tooltip">/sˠawɾˠəɣ/</span></span> (Summer)
  - **IPA Translation:** `/ˈsˠəu.ɾˠə/` (Pronounced roughly like Sow-ruh)

## How FSTs Automate Pronunciation

If we were to teach a human how to pronounce <span class="ga-word">Samhradh<span class="ga-tooltip">/sˠawɾˠəɣ/</span></span>, they would have to
memorize that the <span class="ga-word">mh<span class="ga-tooltip">/w/</span></span> in the middle of a word often sounds like a "w" or "v",
and that the <span class="ga-word">adh<span class="ga-tooltip">/aɣ/</span></span> at the end is often swallowed into a short "uh" sound.

An FST automates this deduction instantly using formal logic. We program the FST
with the strict phonetic rules of the Irish language (like <span class="ga-word">caol le caol<span class="ga-tooltip">/kɪːlˠ lʲɛ kɪːlˠ/</span></span>):

1.  **The Input Tape:** The FST reads the literal string `S a m h r a d h`.
2.  **The Rulebook:** It rapidly checks the context of every letter. It sees the
    `m` followed by an `h`, realizes they act as a single linguistic unit (a
    digraph), and looks at the surrounding vowels to determine if it should
    produce a "broad" or "slender" sound.
3.  **The Output Tape:** As it processes these rules, it writes the
    corresponding, unambiguous IPA symbols to the output tape: `/ˈsˠəu.ɾˠə/`.

### Why this Matters

By mapping the entire phonological rulebook of Irish into an FST, Aithesc.io can
look at **any\*** Irish word—even words it has never seen before—and perfectly
generate the correct IPA pronunciation.

This string-rewriting technology bridges the crucial gap between text and
speech. It is the foundational technology required to build accurate
text-to-speech (TTS) systems, digital pronunciation guides, and interactive
tools that finally take the guesswork—and the comedian's punchlines—out of
learning the Irish language.

## Building a Transducer with Pynini

To show how this string rewriting actually works under the hood, let's look at a
highly simplified example using **Pynini**, the Python library Aithesc.io uses
to build Finite State Transducers.

Imagine we want to build a tiny FST that specifically translates the literal
Irish word "bád" (boat) into its IPA pronunciation `/bˠaːd̪ˠ/`.

First, we import the Pynini library so we have access to its transducer tools:

```python
import pynini
```

Next, we define our alphabet (the characters the machine is allowed to read or
write). In formal language theory, this is often called the "sigma" (Σ) or
universal character set:

```python
sigma = pynini.union(*"[a-zA-Záéíóúˠːd̪/]")
```

Now, we create the core rewriting rule. The `cross` function tells the FST:
"Whenever you see the exact string on the left, output the exact string on the
right." Here, we are crossing the literal spelling "bád" with the IPA string
"/bˠaːd̪ˠ/":

```python
pronunciation_rule = pynini.cross("bád", "/bˠaːd̪ˠ/")
```

If we want to test our new machine, we can take an input string (like "bád") and
push it through our rule using the `compose` function (meaning, push the input
through the transducer network):

```python
input_string = pynini.accep("bád")
result = pynini.compose(input_string, pronunciation_rule)
```

Finally, we tell Python to print out the result of the rewrite. If everything
worked correctly, the output tape will show the IPA:

```python
print('/ ' + result.string() + ' /')
# Output: / bˠaːd̪ˠ /
```

While this example only handles a single word, the real Aithesc.io phonetics
engine works the exact same way. It simply scales this up, replacing literal
words with massive dictionaries of dynamic rules (like "if you see an 'm' next
to an 'h' around slender vowels... rewrite it to an un-rounded 'v'").

## Advanced Example: Contextual Rewriting

While the previous example crosses an exact word, the true power of an FST lies
in **context-dependent rewriting**. This allows us to write broad rules that
apply to any word, provided specific conditions are met.

Let's build a rule to handle the Irish word "mbualadh" (their striking). This
word features two complex phonetic translations:

1.  **Eclipsis:** The initial `b` is eclipsed by `m`, meaning the `b` becomes
    completely silent. The `mb` cluster is simply pronounced `/m/`.
2.  **Multi-Letter Suffix:** The ending `-adh` is collapsed into a single short
    vowel sound, the schwa `/ə/`.

To parse this dynamically, we use Pynini's `cdrewrite` (context-dependent
rewrite) function.

First, we define a rule that says "whenever you see the string 'mb', rewrite it
as 'm', but ONLY if it sits at the **very beginning** of the word." If "mb"
appears anywhere else, we leave it alone. We specify this by requiring the Left
Context to be `[BOS]` (Beginning of String):

```python
# The .closure() on the alphabet allows any combination of valid letters
sigma_star = pynini.union(*"[a-zA-Záéíóúə/]").closure()

# Cross 'mb' to 'm', ONLY at the start of the word
eclipsis_b = pynini.cdrewrite(
    pynini.cross("mb", "m"),
    "[BOS]",  # Left context (must be at the start of the word)
    "",  # Right context (empty = anywhere)
    sigma_star
)
```

Next, we define a rule that collapses the complex multi-letter suffix `-adh`
into the single phoneme `/ə/`. However, we only want this to happen if the
letters appear at the **very end** of a word, not in the middle. We specify this
by requiring the Right Context to be `[EOS]` (End of String):

```python
# Cross 'adh' to 'ə', but ONLY if it sits right before the End of String ([EOS])
suffix_adh = pynini.cdrewrite(
    pynini.cross("adh", "ə"),
    "",       # Left context (empty = anywhere)
    "[EOS]",  # Right context (must be at the end of the word)
    sigma_star
)
```

Finally, we chain these rules together. FSTs can be "composed" in sequence using
the `@` operator, meaning the output tape of the first rule is fed directly into
the input tape of the second rule.

```python
# Compose the rules sequentially into a mini-engine
irish_phonetics_engine = eclipsis_b @ suffix_adh

# Feed the complex word "mbualadh" into our new engine
input_string = pynini.accep("mbualadh")
result = pynini.compose(input_string, irish_phonetics_engine)

print('/ ' + result.string() + ' /')
# Output: / mualə /
```

_(Note: To get the completely accurate IPA `/mˠuəl̪ˠə/` with broad consonants, we
would simply chain a few more vowel-context rules to our
`irish_phonetics_engine`!)_

By chaining many of these highly specific, context-dependent rewrite rules
together, an FST mathematically translates any Irish text into perfect phonetic
notation, elegantly handling every mutation and multi-vowel suffix along the
way.
