# Project Proposal: Neuro-Symbolic Integration of LLMs and FSTs for Irish Morphological Parsing

## 1. Executive Summary

This project proposes the development of a **Neuro-Symbolic architecture** for processing Modern Irish. By combining the probabilistic, semantic reasoning capabilities of Large Language Models (LLMs) with the strict, deterministic morphological and morphotactic rules encoded in Finite-State Transducers (FSTs) via the Pynini library, we aim to solve the critical "hallucination and grammatical accuracy" problem that plagues deep learning models when dealing with low-resource, highly inflected languages.

## 2. The Problem: LLMs and Under-Represented Languages

Recent research highlights a severe limitation in current Large Language Models: they struggle fundamentally with **morphologically rich, low-resource languages** like Irish. 

1.  **Data Sparsity & Hallucination:** Because major LLMs are trained primarily on high-resource languages (English, Spanish, Mandarin), the Irish training data they encounter is sparse and often noisy. When asked to generate Irish, LLMs frequently "hallucinate" non-existent morphological forms or apply English-centric subword tokenization that destroys the Celtic root structure.
2.  **Morphological Complexity:** Irish features intricate initial consonant mutations (eclipsis and lenition), complex verb paradigms, and stem allomorphy. LLMs, which rely on statistical pattern matching, often fail to generalize these strict, interdependent grammatic rules when data is lacking, resulting in grammatically awkward or culturally insensitive outputs.
3.  **Cross-Lingual Interference:** Multilingual LLMs often misapply grammatical patterns from higher-resource languages onto Irish (e.g., failing to mutate a noun after a preposition because such rules don't exist in English).

## 3. The Baseline: Finite-State Transducers (FSTs)

Conversely, traditional Computational Linguistics has successfully modeled Irish morphology using Finite-State Transducers (FSTs). Researchers like Elaine Uí Dhonnchadha <cite>[UiDhonnchadhaMorphology]</cite> and Kevin Scannell <cite>[ScannellStandardiser]</cite> have proven that FSTs can accurately encapsulate the entire morphotactic rule system of Irish vocabulary. 

*   **Pros:** FSTs provide 100% accuracy for known lexical items. They perfectly handle bidirectional processing (parsing a surface form down to its lemma, or generating a mutated surface form from a lemma + tags).
*   **Cons:** FSTs are purely symbolic. They cannot understand semantic context, cannot easily guess the meaning of out-of-vocabulary (OOV) words, and generate massive ambiguity (e.g., returning 5 different valid parses for a single word, requiring external Constraint Grammars to disambiguate).

## 4. The Proposed Solution: A Neuro-Symbolic Architecture

We propose integrating an FST-based morphological engine (built in Python using **Pynini**) directly into an LLM pipeline. This **Neuro-Symbolic** approach leverages the strengths of both paradigms across the entire language-processing lifecycle:

*   **The "Symbolic" Component (Pynini FST):** Acts as the foundational morphological layer for both input and output. During input analysis, it parses raw Irish text into ambiguous lemmas and tags. During output generation, it acts as a deterministic "Grammar Guardrail," transducing abstract linguistic representations back into perfectly mutated surface forms (e.g., `mbád`).
*   **The "Neural" Component (LLM):** Handles semantic reasoning, translation intent, and context. On the input side, it receives the multiple possible parses generated by the FST and disambiguates them based on sentence context. On the output side, rather than generating raw Irish text directly, the LLM is prompted to produce **Abstract Morphological Representations** (e.g., `[LEMMA: bád, POS: Noun, CASE: Genitive, NUMBER: Plural, MUTATION: Eclipsis]`) for the FST to compile.

### 4.1. Core Pipeline

1.  **Analysis (Input):** Irish text is tokenized. The Pynini FST parses the text into all possible lemmas and morphological tags.
2.  **Neural Disambiguation:** The LLM receives the ambiguous FST output and uses the surrounding semantic context to select the correct parse (replacing legacy Constraint Grammar).
3.  **Reasoning/Translation:** The LLM performs the core NLP task (summarization, translation, QA) in the abstract semantic space.
4.  **Generation (Output):** The LLM outputs a series of lemmas and desired grammatical tags. The Pynini FST transduces this abstract output back into grammatically flawless Modern Irish surface text.

## 5. Value Add and Broader Impact

This project holds significant value specifically for under-represented and Celtic languages:

*   **Data Efficiency:** Neuro-symbolic models require drastically less training data to achieve high grammatical accuracy because the grammar is explicitly programmed into the FST, rather than statistically inferred by the LLM over millions of examples. 
*   **Explainability & Trust:** Unlike purely neural "black box" generators, if a grammatical error occurs, developers can pinpoint exactly whether the LLM chose the wrong tag or the FST had a missing rule, allowing for immediate, targeted fixes.
*   **A Blueprint for Low-Resource NLP:** Successfully demonstrating this architecture for Irish provides a scalable blueprint for other digitally endangered or morphologically complex languages globally, proving that we do not need to wait for massive datasets to arrive before these languages can benefit from the AI boom.

## 6. Next Steps

1.  **Prototype Phase:** Develop a small-scale Pynini transducer for a subset of Irish morphology (e.g., noun mutations and basic declensions).
2.  **Integration Phase:** Build a Python interface linking the Pynini FST with a local, open-weights LLM (e.g., Llama 3 or Mistral) via an orchestration framework like LangChain.
3.  **Evaluation Phase:** Compare the output quality of the standalone LLM against the Neuro-Symbolic pipeline using synthetic data and established Irish treebanks.
