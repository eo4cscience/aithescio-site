# Modeling with Finite State Transducers

## The Necessity of Using the Linguistic Theory Pipeline

Irish morphology is highly fusional and features complex interactions between
phonology and syntax. Attempting to build a monolithic FST would result in
unmanageable state explosion and logic errors.

The way to handle a complex task like that is to carve it up in smaller pieces
and then integrate them into an overall pipeline. For the processing of
languages the typical approach is to break the system down into layers that
reflect the natural linguistic structure of the language. Each layer can be
developed and tested independently, ensuring that the rules of one layer do not
interfere with those of another.

Using the finite state transducer method we can create separate transducers for
each linguistic process, such as one for the lexicon, another for processing
the lexicon, handling morpho-tactics, and a another for morpho-phonology. This
modular design allows us to focus on the specific rules and exceptions of each
layer without worrying about the combinatorial explosion that would occur if we
tried to model everything in a single transducer.

The way in which these stages are integrated into a single pipeline is by using
the mathematical operation of **composition**. Using this operation we feed the
output from one transducer into the input of the next. Each transducer can be
thought of as a function that takes an input and produces an output. By
composing these functions, we can create ultimately create a single transducer
that takes the initial input (e.g., a lemma and set of tags) and produces the
final output (the surface form) while ensuring that all the rules of each layer
are applied correctly.

The composition functcion has its own operator and the pipeline is defined as
follows: `Lexicon_fst ∘ Morphotactics_fst ∘ Morphophonology_fst`. We create a
single, efficient engine that preserves the rigorous rules of each layer while
functioning as a cohesive system. This modular approach ensures that updates to
the lexicon do not break the underlying phonological rules.

## Bidirectional Processing: Analysis and Generation

The utility of a computational morphological model is defined by its capacity to function across two distinct linguistic vectors: decoding and encoding. From a computational perspective, we require a system that facilitates the 'understanding' of ingested natural language text through precise decomposition, the analytic pipeline, as well as a system capable of 'creating' or synthesizing text to communicate intent, the generative pipeline. The integration of both pipelines is essential for any bidirectional linguistic application, ensuring that the formal structures of the language can be both retrieved from surface tokens and projected from abstract grammatical representations.

Here is where the power of Finite State Transducer modeling lies. The power of Finite State Transducer modeling lies in its inherent mathematical symmetry, which allows a single composed pipeline to be traversed in two
diametrically opposed directions without requiring separate logic for each. In
our development, we prioritize the **generative direction** (intent → surface)
as our primary engineering path. This is because we can work from a very stable
and well-defined base form, the lexicon and grammatical rules, to precisely
control the expansion into complex surface forms. By building the generative
pipeline first, we establish a "source of truth" that ensures every word
produced is linguistically valid.

However, the ultimate goal of the project is often the **analytic pipeline**
(surface → intent), which allows a computer to "read" and parse existing Irish
text. To achieve this, we do not build a second parser from scratch. Instead, we
perform the mathematical operation of **inversion**. Since an FST is essentially
a mapping between two sets of strings, inverting the transducer simply swaps the
input and output labels on every arc. This instantaneously converts our
generative model into an analytic one, allowing us to leverage the same rigorous
rules for both production and analysis.

In **Pynini**, this inversion is handled elegantly with a single function call.
Once the generative pipeline is composed, it can be flipped to create the
analyzer:

```python
import pynini

# Define the generative pipeline (Grammar → Surface)
grammar_to_surface = (lexicon @ morphotactics @ morphophonology).optimize()

# To progress to the analytic pipeline (Surface → Grammar), we invert the FST
surface_to_grammar = pynini.invert(grammar_to_surface)

# Usage
# Generation: intent.optimize() @ grammar_to_surface
# Analysis:   surface.optimize() @ surface_to_grammar

# In this case intent and surface are the input strings
# for generation and analysis respectively.
```

This bidirectionality ensures that the system serves not just as a lookup table,
but as a complete, two-way computational gateway between the abstract structure
of the Irish language and its surface-level expression.

## FST Modeling Staging Sequence

As we explained above, we need to create a multi-stage pipeline to handle the
full scope of the Irish linguistics. Each stage corresponds to a specific
linguistic layer, and the output of one stage serves as the input for the next.
This structured approach allows us to manage the complexity of Irish morphology
while ensuring that each layer is built on a solid foundation.

The project follows a strict sequential development path to manage the inherent
complexity of Irish grammar, ensuring each layer of the FST provides a stable
foundation for the next.

The architectural decision to sequence the development of the FST from the Lexicon through Morphotactics to Morphophonology is grounded in the principle of increasing computational volatility. We begin with the **Lexicon**, the most stable layer, which serves as the static repository of lemmas. For instance, the entry for the noun _cat_ (cat) remains constant regardless of the grammatical environment. This stage is primarily a data-modeling exercise, making it the most straightforward to implement as it relies on firm, declarative entries. 

Once the lexical foundation is laid, we proceed to **Morphotactics**, where we define the formal rules of affixation and inflection. Here, the rules remain relatively stable but introduce the first layer of structural complexity; for example, defining that the plural of _cat_ necessitates a specific path to a plural suffix or stem change (_cait_). Finally, we implement the **Morphophonology** layer, which is the most difficult and unstable segment of the pipeline. Unlike the previous stages, morphophonology is context-sensitive and requires complex rewrite rules (such as Pynini's `cdrewrite`) to handle initial consonant mutations—like the lenition of _cat_ to _chat_—and orthographic harmony rules like _Caol le Caol_. Because these rules are subject to the highest degree of contextual interference and phonetic "drift," they represent the most challenging logic to capture with deterministic precision. By building from the stable core outward to the volatile surface, we ensure that the structural integrity of the linguistic model is preserved at every level of composition.

**Morphotactics: Nominal Paradigm** defines the lexicon of nouns and their
corresponding declension classes. This stage involves mapping root stems to
their various inflected forms across case, number, and gender. It is required to
handle the high degree of fusion in Irish nouns, where a single suffix might
simultaneously indicate gender and case. These nominal FSTs will eventually be
composed with the phonological layers to ensure that a genitive plural suffix,
for example, triggers the correct internal vowel shifts and initial mutations.

**Morphotactics: Verbal Paradigm** addresses the most computationally dense area
of Irish morphology, covering the intricate rules of tense, aspect, mood, and
person. This stage is critical for capturing the distinction between synthetic
forms (where the subject is fused into the verb) and analytic forms, as well as
the unique stems of the eleven irregular verbs. Because verbs are the anchors of
Irish syntax, this layer serves as the primary engine for sentence generation,
providing the necessary inflected forms that preverbal particles will later act
upon.

**Closed Classes & Functional Morphemes** handles the "grammatical glue" of the
language, specifically focusing on prepositional pronouns and the mutation
triggers caused by articles and numerals. This stage is necessary to resolve
high-frequency functional items that do not fall into standard noun or verb
categories but are essential for basic sentence construction. Once implemented,
these transducers allow the system to process the complex fusions characteristic
of Celtic languages, such as the merging of prepositions with personal pronouns.

**Morphophonology & Phonotactics** represents the most granular layer of the
system, responsible for handling character-level changes such as initial
consonant mutations (lenition and eclipsis) and orthographic harmony (the _Caol
le Caol_ rule). This stage is essential because it defines the "physics" of the
Irish writing system; it acts as the final word-level processor that ensures
all previous structural shifts (from nouns, verbs, or closed classes) result in
valid, harmonized surface forms.

**Sentence-Level Alignment** marks the final integration phase where the FSTs
are composed to process full strings and handle long-distance syntactic
dependencies. This stage is needed to manage "top-down" grammatical triggers,
such as the specific mutations required by relative clauses or preverbal
particles like _ní_ and _go_. By completing this sequence, the FST transitions
from a word-level analyzer to a complete computational grammar that can serve as
a source of absolute truth for downstream LLMs or linguistic research tools.

## The Full Connection: From Lexicon to Surface

To illustrate this clearly, let's use the Irish feminine noun bróg (shoe). We will put it into the genitive case (meaning "of the shoe," as in "the color of the shoe"), which requires both a morphological suffix and a phonotactic adjustment.

### The Pipeline for An Bhróige (Of the shoe)

**Input (Lexical Stage)**:The system starts with the root and the grammar tags:Art + bróg + Noun + Fem + Sing + Gen

**Morphophonology (Early Stage)**:The grammar rules for the feminine genitive singular with the article an trigger an initial mutation (Lenition).Rule: b → bh Result: An bhróg...

**Morphology (Suffixation)**:The genitive case for this class of nouns requires the addition of the suffix -e.Intermediate Form: An bhróg + e → An bhróge (This is "illegal" in Irish).

**Phonotactics (Late Stage - The "Double Change")**:The phonotactic rule Caol le caol (Slender with slender) now acts as a mandatory filter. Because the suffix -e is a slender vowel, the preceding consonant g must also become slender (palatalized) to be "legal.
* "Constraint: A broad consonant (velarized) cannot immediately precede a slender vowel.
* Adjustment: To show the g is now slender, a slender vowel (i) must be inserted before it to "bridge" the sound.
* Surface Form: An bhróige
