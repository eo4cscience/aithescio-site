# Nominal Paradigms: The 2nd Declension

## Introduction
While the previous chapter focused on the verbal system, Pynini's `Paradigm` class is equally adept at modeling the complex nominal systems of Irish. Nouns in Irish are grouped into five primary declensions, each with distinct patterns for case and number.

## Nominal Feature Space and Tags

In the Pynini implementation, the nominal grammatical space is defined by a set of feature-value pairs. These tags permit the system to uniquely identify the morphosyntactic properties of any given noun form.

| Feature | Code | Description | Valid Values |
| :--- | :--- | :--- | :--- |
| **Case** | `case` | The grammatical case of the noun. | `nom`, `gen`, `voc`, `dat` |
| **Number** | `number` | The grammatical number. | `sg`, `pl` |
| **Gender** | `gender` | The biological or lexical gender. | `masc`, `fem` |
| **Declension** | `decl` | The inflectional class of the noun. | `D1`, `D2`, `D3`, `D4`, `D5` |

Each unique intersection in this space (e.g., `case=gen` × `number=sg` × `gender=fem`) represents a "cell" in the paradigm that is mapped to a specific FST operation in the `slots = []` variable.


To illustrate nominal modeling, we will examine the **2nd Declension**. This declension primarily consists of feminine nouns that form their genitive singular by adding the suffix `-e` and slenderizing (palatalizing) the final consonant.

## Modeling the 2nd Declension

The 2nd declension provides an excellent example of **morphophonological epenthesis**. When a broad stem like *cloch* (stone) becomes slender in the genitive, the spelling system inserts an `i` before the final consonant to mark the shift in quality.

### 1. Feature Definition
We define the nominal categories for Case and Number.

```python
from pynini.lib import features, paradigms

case = features.Feature("case", "nom", "gen", "voc", "dat")
number = features.Feature("number", "sg", "pl")
noun_cat = features.Category(case, number)

# Lemma: Nominative Singular
lemma = features.FeatureVector(noun_cat, "case=nom", "number=sg")

# Target: Genitive Singular
gen_sg = features.FeatureVector(noun_cat, "case=gen", "number=sg")
```

### 2. Slenderization Rules
Unlike the verbal example where we used a simple vowel rewrite, the nominal genitive requires **slenderization**—the transition of a broad consonant to a slender one. We model this by inserting the "bridge" vowel `i`.

```python
# Rule: Insert 'i' before the consonant cluster in the genitive environment (+E)
slenderize = pynini.cdrewrite(
    pynini.cross("", "i"), 
    broad_vowels, 
    consonants.plus + "+E", 
    sigma_star
)

# Clean up the virtual marker and add the suffix vowel 'e'
finalize = pynini.cdrewrite(pynini.cross("+E", "e"), "", "", sigma_star)

nominal_rules = (slenderize @ finalize).optimize()
```

### 3. Compiling the Paradigm
The `Paradigm` object integrates these rules with the morphological slots.

```python
slots = [
    (paradigms.suffix("", stem), lemma),
    (paradigms.suffix("+E", stem), gen_sg)
]

irish_nouns = paradigms.Paradigm(
    category=noun_cat,
    slots=slots,
    lemma_feature_vector=lemma,
    stems=["cloch", "bróg"],
    rules=[nominal_rules]
)
```
