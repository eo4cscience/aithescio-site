# Pipeline Walkthrough: A Minimal Example

To understand how the Irish FST pipeline functions, it is helpful to trace a single word through its entire transformation. In this example, we will generate the phrase *mo chat* (my cat) starting from a raw lexical intent.

The mathematical pipeline is defined as:  
`Lexicon ∘ Morphotactics ∘ Morphophonology = Surface`

---

## 1. The Lexicon Layer
**Input:** `mo` (my) + `cat` (cat)  
The Lexicon FST maps the raw input into a standardized "Deep String" that the rest of the engine can understand.

```python
import pynini

# Simplified Lexicon Mapping
lexicon = pynini.string_map([
    ("mo", "mo<POSS>"),
    ("cat", "cat<N><MASC><SG>")
])

# Resulting Deep String:
# "mo<POSS> cat<N><MASC><D1><SG>"
```

---

## 2. The Morphotactic Layer
**Input:** `mo<POSS> cat<N><MASC><SG>`  
The Morphotactic FST identifies that the possessive pronoun *mo* triggers lenition on the following noun. It inserts an **abstract trigger tag** (`^LEN`) to instruct the next layer.

```python
# Rule: If preceded by mo<POSS>, insert ^LEN before the noun stem
# pynini.cross("", "^LEN") is a transducer that changes "" into "^LEN"
morphotactics = pynini.cdrewrite(
    pynini.cross("", "^LEN"),
    "mo<POSS> ",
    "cat<N><MASC><D1><SG>",
    pynini.union(*"abcdefghijklmnopqrstuvwxyz").closure()
)
```
*Note: Since the cross() function is inside pynini.cdrewrite(), the change is context-dependent.*
# Resulting Intermediate String:
# "mo cat^LEN"
```
*Note: The grammatical tags like `<POSS>` and `<N>` are typically consumed or stripped at this stage once their logic has been applied.*

---

## 3. The Morphophonological Layer
**Input:** `mo cat^LEN`  
The Morphophonology FST is "blind" to the fact that this is a possessive construction. It only see the instruction `^LEN` and applies the phonetic "physics" of Irish.

```python
# Rule: Replace 'c' with 'ch' when followed by the ^LEN trigger
# pynini.cross("c", "ch") changes the 'c' into a 'ch'
lenition_rule = pynini.cdrewrite(
    pynini.cross("c", "ch"),
    "",
    "^LEN",
    pynini.union(*"abcdefghijklmnopqrstuvwxyz").closure()
)

# Rule: Clean up by removing the trigger tag
cleanup = pynini.cdrewrite(
    pynini.cross("^LEN", ""),
    "",
    "",
    pynini.union(*"abcdefghijklmnopqrstuvwxyz").closure()
)

morphophonology = pynini.compose(lenition_rule, cleanup)

# Resulting Surface String:
# "mo chat"
```

---

## The Final Composition
By composing these three FSTs <span class="cite-hover-static">[Roark2007]<span class="cite-tooltip-static"> (2007). <i>Computational approaches to morphology and syntax</i>.</span></span>, we create a single engine that performs the entire transformation in one pass.

```python
# The complete generative pipeline
pipeline = lexicon @ morphotactics @ morphophonology

# Generation
print(pynini.compose("mo cat", pipeline).string())
# Output: "mo chat"

# Analysis (Inversion)
analyzer = pynini.invert(pipeline)
print(pynini.compose("mo chat", analyzer).string())
# Output: "mo cat"
```

This walkthrough demonstrates the clean **separation of concerns**: the Lexicon handles data, Morphotactics handles the "why" (grammar), and Morphophonology handles the "how" (sounds/spelling).
