# Lexicon

The **Lexicon FST** is the very first transducer in the generative pipeline and serves as the bridge between the raw linguistic data and the formal computational grammar. Its primary role is to act as a translation layer, ensuring that the information stored in our lexicon files is correctly formatted for the downstream linguistic FSTs.

## The Role of the Lexicon FST

In a generative pass, the system starts with a **base lexeme** (the root word) and a set of **grammar rules** (represented by tags). However, we cannot assume that the raw entries in our lexicon will naturally match the specific "pattern strings" that our FSTs expect. For instance, a dictionary entry might store a noun and its gender as separate fields, but the FST requires a single, unified string like `cloch<cite-hover citekey="noun">[noun]</cite-hover><cite-hover citekey="fem">[fem]</cite-hover><cite-hover citekey="plural">[plural]</cite-hover>`.

The Lexicon FST extracts the raw information from the lexicon file and converts it into this precise, predictable form. Because this is the first stage in the pipeline, it ensures that every subsequent stage—from Morphotactics to Morphophonology—receives a perfectly sanitized input.

## How it Works: From Raw Data to FST Input

Think of the Lexicon FST as a specialized filter. It maps the idiosyncratic format of a dictionary or database into the "deep" representation used by the transducer engine.

### Example: Noun Inflection
Suppose we have a raw entry for the Irish word for "stone":
- **Lemma:** `cloch`
- **Class:** `Noun`
- **Gender:** `Feminine`

If the user (or a downstream AI) intends to generate the plural form, the Lexicon FST takes these components and produces the following target string:

**Raw Intent:** `cloch` + `plural`
**Lexicon FST Output:** `cloch<noun><fem><D2><pl>`

In this output, the `<D2>` tag explicitly identifies the word as belonging to the **Second Declension**. This is a critical architectural requirement: since the ending and gender of a word are not always enough to uniquely determine its inflectional class, the Lexicon FST must provide this specific "instruction" to the downstream Morphotactic layer.

The **Morphotactic FST** then recognizes both the `<D2>` and `<pl>` tags and knows exactly which set of state transitions to follow, resulting in the intermediate form `clocha`.

### Example: Verb Conjugation
For a verb like *glan* (to clean):
- **Lemma:** `glan`
- **Class:** `Verb`
- **Conjugation:** `1st`

**Raw Intent:** `glan` + `past` + `1st person singular`
**Lexicon FST Output:** `glan<verb><past><1sq>`

By standardizing these inputs at the very beginning, we decouple the "storage" of the Irish language (the lexicon) from the "logic" of the Irish language (the FST rules). This allows researchers to update the lexicon with new words without ever needing to modify the complex mathematical state machines that handle the actual grammar.

## Summary of the Pipeline Start
1.  **Input:** User/System selects a root lexeme and a set of desired grammatical features.
2.  **Lexicon FST:** Maps the root and features into a unified "Deep String" (e.g., `stem<tag1><tag2>`).
3.  **Output:** The Deep String is ready to enter the **Morphotactic Layer**, where the actual construction of the word begins.
